Les personnages

  • Legacy : le sage et sénile
    il est pénible, coûte cher et rechigne à évoluer
  • Symfony : l'apprenti prometteur
    sympa et rapide, mais il ne sait pas encore tout faire

Objectif : retraite

on ne peut pas remplacer le legacy d'un coup

⇒ strangler pattern

Strangler pattern - avec isolation

Symfony et Legacy ne se parlent pas

utile par exemple si Legacy pas en php ou dans une très vieille version

Symfony renvoi vers url dédiée au legacy s'il ne sait pas traiter

Usage partiel du Legacy possible par appel http côté client (ajax)

Strangler pattern - avec couplage

Ils sont rassemblés dans une seule application

Si Symfony ne sait pas traiter, il boot le legacy et lui laisse la main

  1. Le legacy peut faire appel à du code Symfony
  2. Le code Symfony ne doit pas utiliser le code legacy

On veut bien contaminer le legacy par du Symfony mais pas l'inverse

Par quels sujets commencer

  • Controlleur capable de boot le legacy
  • Auth
  • Load des paramètres
